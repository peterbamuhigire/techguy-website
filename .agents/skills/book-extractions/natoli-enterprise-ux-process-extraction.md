# Natoli - Enterprise UX Process

**Source:** Joe Natoli, *Fixing the Enterprise UX Process*.

## Use this for

- Product discovery
- Feature-definition workflows
- Cross-functional UX process
- Requirements generated from prototype evidence

## High-signal insights

1. Stop jumping to solutions. The fastest enterprise teams do not start with features; they start with context, constraints, and the exact task users are trying to complete.
2. UX is everybody's business. Discovery quality collapses when design, product, engineering, and stakeholders work sequentially instead of in the same room.
3. Replace heavy personas with fast context. One-page summaries, contextual scenarios, and direct interview notes beat ornate persona decks.
4. Prototype early to generate requirements. Low-fidelity flows reveal missing requirements faster than meetings and speculative user stories.
5. Separate task completion from success. "User finished the flow" is weaker than "user reached the intended business and user outcome."
6. If a feature remains undefined after two sprints, table it or kill it.

## Encode into skills

- Use discovery artifacts as evidence gates, not decoration.
- Require prototype validation before detailed requirements.
- Write user stories only after the team understands the scenario and the action model.
- Force explicit go / refine / drop decisions on fuzzy features.
