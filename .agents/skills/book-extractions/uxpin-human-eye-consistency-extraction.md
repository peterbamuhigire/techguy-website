# UXPin - Human Eye Visual Consistency

**Source:** UXPin, *Web UI Design for the Human Eye: Principles of Visual Consistency*.

## Use this for

- Visual systems
- Pattern governance
- Design audits
- Component consistency

## High-signal insights

1. Research expectations before design. Designers bias themselves when they start drawing before checking what users already expect from the category.
2. Consistency is not sameness. Preserve a stable logic for labels, controls, spacing, and state changes while leaving room for brand character.
3. Affordances and signifiers are foundational. If a control looks decorative, users hesitate or misfire.
4. Pattern libraries reduce cognitive cost only when teams apply them consistently. Pattern sprawl becomes UX debt.
5. Consistency must hold across layout, language, spacing, icon treatment, and interaction feedback, not just color.
