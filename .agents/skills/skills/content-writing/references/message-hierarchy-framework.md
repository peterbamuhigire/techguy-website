# Message Hierarchy Framework

Use this reference when turning strategy into page messaging.
It combines StoryBrand clarity, Dunford-style positioning specificity,
They Ask, You Answer transparency, and Made to Stick memorability.

## Core Rule

Every important page should answer these questions in order:

1. Who is this for?
2. What problem are they trying to solve?
3. Why does this brand understand that problem?
4. What is different here?
5. What proof makes the claim believable?
6. What should the visitor do next?

## Message Hierarchy

### 1. Audience Signal

Lead with the audience or use case when helpful.

Examples:

- `For growing firms that need better-fit leads`
- `For law practices that need trust before contact`

### 2. Problem

Frame the problem in the buyer's language.

Capture:

- external problem
- internal frustration
- high-level stakes

### 3. Promise

State the transformation simply.

Good promise test:

- concrete
- business-relevant
- free of puffed-up adjectives

### 4. Differentiator

State what makes the approach or offer distinct.
This is where Dunford's value themes and market frame should surface.

### 5. Proof

Back the claim with specifics:

- results
- credentials
- examples
- process
- comparison clarity

### 6. Call To Action

Use one direct CTA and one softer transitional CTA where needed.

## Sticky Message Check

Use the SUCCESs lens:

- Simple: is the core idea obvious?
- Unexpected: is there a tension, contrast, or reframe?
- Concrete: are we using specifics instead of abstractions?
- Credible: what makes this believable?
- Emotional: why should this matter to the buyer?
- Story: can the reader imagine the before and after?

## "They Ask" Check

If a page or section avoids the real buyer question, it is incomplete.

Explicitly answer:

- cost
- fit
- tradeoffs
- problems or limitations
- comparisons
- best options

## Common Failure Modes

- headline is broad but says nothing
- proof appears too late
- CTA asks for more commitment than the message has earned
- page sounds polished but not memorable
- message describes the company, not the buyer's progress
