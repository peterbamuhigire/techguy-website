# CRO Heuristic Checklist

A structured pass/fail audit for evaluating any website's conversion readiness.
Mark each item Pass, Fail, or N/A. Count failures per section. Use failures to populate the Top 10 Issues list in the audit report.

Sources: Brunson split test data; Nahai (Cialdini's six principles, Fogg credibility, Paradox of Choice, Hick's Law); Murtagh (22 mistakes, Conversion Pyramid, 3-click rule, 1-click conversion rule).

---

## Section 1: Clarity

Can a stranger understand what the site offers within 5 seconds, without any prior context?

| # | Check | Result |
|---|-------|--------|
| 1.1 | The primary headline communicates a specific outcome or benefit for the visitor (not the company's name, tagline, or a generic welcome message) | |
| 1.2 | A visitor who has never heard of the brand can state what the site sells or does within 5 seconds of landing | |
| 1.3 | The value proposition is stated in terms of what changes for the customer, not what features the product has | |
| 1.4 | There are no industry jargon terms in the hero section that an ideal customer would not immediately understand | |
| 1.5 | The headline is written in present tense, not future conditional ("gives you" not "will give you") | |
| 1.6 | The site's primary purpose (buy, sign up, book, contact) is unambiguous from the landing page | |
| 1.7 | There is no "welcome to our website" or equivalent throat-clearing copy anywhere | |

Section score: __ / 7 pass

## Section 2: Above the Fold

Every element that appears before scrolling is evaluated here.

| # | Check | Result |
|---|-------|--------|
| 2.1 | The hero section contains all five elements: headline, subhead, primary CTA, supporting image, and at least one trust signal | |
| 2.2 | The headline contains a specific number or concrete reference rather than a round number or vague claim | |
| 2.3 | No human face is placed directly beside the headline (faces must face toward copy, not at the camera or away) | |
| 2.4 | The primary CTA button is visible above the fold on both desktop (1280px) and mobile (375px) | |
| 2.5 | The hero image or background does not compete with the headline for visual dominance | |
| 2.6 | The above-fold section is free of navigation-level clutter, multiple competing CTAs, or information overload | |
| 2.7 | The page loads visual content above the fold in under 2 seconds on a standard connection | |
| 2.8 | On mobile (375px), the primary CTA is reachable with the thumb without horizontal scrolling | |

Section score: __ / 8 pass

## Section 3: Navigation and Information Architecture

| # | Check | Result |
|---|-------|--------|
| 3.1 | Primary navigation contains 5-7 items maximum (Hick's Law; Paradox of Choice) | |
| 3.2 | Navigation labels use customer-language terms, not internal department names | |
| 3.3 | No piece of content is more than 3 clicks from the home page (Murtagh's 3-click rule) | |
| 3.4 | Every page on the site includes at least one conversion mechanism — a CTA, opt-in, contact link, or booking link — directly clickable from that page (Murtagh's 1-click conversion rule) | |
| 3.5 | The most important navigation item is accessible from the top-left area of the page | |
| 3.6 | There are no hover-only navigation menus (mobile-incompatible; accessibility failure) | |
| 3.7 | The footer contains secondary navigation and basic contact information | |
| 3.8 | The breadcrumb trail or URL structure makes it clear where the visitor is within the site at all times | |
| 3.9 | There is no dead end — every page has a next step for the visitor | |

Section score: __ / 9 pass

## Section 4: Social Proof

| # | Check | Result |
|---|-------|--------|
| 4.1 | Testimonials are present on the primary conversion pages (home, services, pricing, checkout) | |
| 4.2 | Testimonials include full names, photos, and specific outcomes or numbers, not vague praise | |
| 4.3 | At least some testimonials are from people who demographically match the target visitor (same role, same problem, same context) | |
| 4.4 | Testimonials are formatted to appear like authentic social media posts rather than polished marketing design (Facebook-style formatting beats standard testimonial design by +296%) | |
| 4.5 | Video testimonials are present on at least the primary sales page (video testimonials double conversion vs. text alone) | |
| 4.6 | The testimonials section is headed by a challenge headline ("Need More Proof?" or equivalent) that pre-empts scepticism | |
| 4.7 | Social proof is placed at the highest-anxiety point in the conversion flow — at or near the order form, booking form, or primary CTA | |
| 4.8 | There is no negative social proof — no "most people don't do X" framing that normalises non-conversion | |
| 4.9 | Third-party credibility indicators are visible: media mentions, accreditations, certifications, awards | |
| 4.10 | Social proof is specific enough to be believable: named person, named outcome, named timeframe | |

Section score: __ / 10 pass

## Section 5: Forms

| # | Check | Result |
|---|-------|--------|
| 5.1 | The form asks only for the minimum information needed at this stage (apply progressive disclosure) | |
| 5.2 | A micro-commitment step precedes the main form ask where technically feasible (survey question, yes-or-no question, or preference selection before email entry) | |
| 5.3 | Field labels are positioned outside the fields, not solely inside as placeholder text | |
| 5.4 | The submit button copy describes what the visitor receives, not the action they perform ("Get my free report" not "Submit") | |
| 5.5 | There is a privacy reassurance line directly below any email field ("We never share your details. Unsubscribe any time.") | |
| 5.6 | Multi-field forms are broken into logical steps (step 1: basic details; step 2: preferences or payment) | |
| 5.7 | Error messages are written in plain language and identify the specific field that needs correction — not generic "form error" messages | |
| 5.8 | There is a clear confirmation state after successful form submission (what happens next, when, how) | |
| 5.9 | For order forms: credit card fields are on the left side of the form | |
| 5.10 | For order forms: Facebook-style testimonials appear at the top of the form, above the offer recap | |
| 5.11 | There is zero empty space on order forms — blank areas are filled with short testimonials | |
| 5.12 | All forms are served over HTTPS | |

Section score: __ / 12 pass

## Section 6: CTAs

| # | Check | Result |
|---|-------|--------|
| 6.1 | Every conversion page has exactly one primary CTA. Secondary CTAs are visually subordinate in size, colour, and placement | |
| 6.2 | CTA buttons are visually distinct from every other element on the page — highest contrast, most prominent colour, largest interactive element | |
| 6.3 | CTA copy uses a direct command, the word "you" or "my," and describes the outcome: "Get my free guide," "Start your trial," "Claim your spot" | |
| 6.4 | CTA buttons do not use weak words: "click here," "submit," "go," "send," "enter" | |
| 6.5 | For free offers, the CTA or its immediate surrounding copy explicitly states "it's free" | |
| 6.6 | For pages with a sales video, the CTA button is below the video (not to the side) | |
| 6.7 | For free trial pages, the primary CTA is visible at the top of the page without scrolling | |
| 6.8 | Order buttons are visible immediately on page load — never hidden until price reveal | |
| 6.9 | CTAs use present-tense language ("Get access now" not "You'll get access") | |
| 6.10 | CTA placement aligns with the terminal point of the page's visual flow (bottom of Z-pattern, end of scroll sequence) | |

Section score: __ / 10 pass

## Section 7: Trust Signals

| # | Check | Result |
|---|-------|--------|
| 7.1 | Presumed credibility: the site's design quality matches or exceeds the standard for its industry category (Fogg: surface credibility is evaluated in milliseconds) | |
| 7.2 | Reputed credibility: third-party endorsements, media logos, awards, or accreditations are visible | |
| 7.3 | Earned credibility: contact information is specific — named team members, physical address, phone number | |
| 7.4 | Security badges (SSL/trust seals) appear only on order forms and checkout pages — not on opt-in pages, registration pages, or blog content | |
| 7.5 | A clear, findable privacy policy exists and is linked from any page requesting personal data | |
| 7.6 | A clear refund or guarantee policy is stated near the primary CTA or order form | |
| 7.7 | Testimonials use full names, photos, specific results, and where possible job titles or company names | |
| 7.8 | The site is completely free of trust destroyers: autoplay audio, aggressive entry pop-ups, outdated copyright year, broken links, generic stock photography | |
| 7.9 | Author or founder photo and biography are present with specific credentials (not vague expertise claims) | |
| 7.10 | Brand voice and visual identity are consistent across all pages — inconsistency signals amateurism and reduces earned credibility | |

Section score: __ / 10 pass

## Section 8: Mobile

| # | Check | Result |
|---|-------|--------|
| 8.1 | All tap targets are at least 44x44px — buttons, links, form fields, and navigation items | |
| 8.2 | Primary CTAs fall within the natural thumb zone (centre to bottom of screen on a 375px device) | |
| 8.3 | Forms display as single-column on mobile with no horizontal scrolling | |
| 8.4 | Headline and primary CTA are visible above the fold on a 375px screen without scrolling | |
| 8.5 | Page load time is under 3 seconds on a simulated mobile connection (Chrome DevTools: Slow 3G simulation) | |
| 8.6 | There is no autoplay video or audio on any mobile view | |
| 8.7 | "Tap to call" links are present on contact pages and in the header for service businesses | |
| 8.8 | The mobile conversion sequence has been tested end-to-end: landing → scroll → form → submit → confirmation | |
| 8.9 | Text is legible without pinching: minimum 16px body text on mobile | |
| 8.10 | There is no content hidden behind hover states (hover does not exist on mobile) | |

## Section 9: Page Speed Signals (Perceptual)

This section audits perceptual performance — what the visitor experiences — rather than technical metrics.

| # | Check | Result |
|---|-------|--------|
| 9.1 | Above-fold content appears visually complete within 2 seconds on a standard broadband connection | |
| 9.2 | Images do not cause layout shift after the page loads (CLS — Cumulative Layout Shift) | |
| 9.3 | Fonts load without a flash of invisible or unstyled text (FOIT/FOUT) | |
| 9.4 | The largest above-fold element (hero image, video thumbnail) loads within 2.5 seconds (LCP — Largest Contentful Paint) | |
| 9.5 | Loading states or skeleton screens are present for any dynamically loaded content | |
| 9.6 | The site feels responsive to interaction — buttons react instantly to taps and clicks | |
| 9.7 | No element moves or re-positions while the visitor is attempting to click or tap it | |

Section score: __ / 7 pass

## Section 10: Psychological Principles

Assess whether each of Cialdini's six principles is present, absent, or overused to the point of damaging trust.

| # | Principle | Present | Absent | Overused / Damaging |
|---|-----------|---------|--------|---------------------|
| 10.1 | Reciprocity: a free, genuinely valuable resource is offered without conditions before asking for anything in return | | | |
| 10.2 | Commitment and Consistency: micro-commitment steps or yes-ladders lead visitors toward the main conversion ask | | | |
| 10.3 | Social Proof: testimonials are prominent, specific, demographically matched, and believably authentic | | | |
| 10.4 | Liking: the site feels warm and relatable; authentic photography; copy in the voice of the ideal customer | | | |
| 10.5 | Authority: specific credentials, accreditations, and expertise signals are visible without vague boasting | | | |
| 10.6 | Scarcity: if used, scarcity is real, specific, and credible — not an obviously artificial countdown that resets on every visit | | | |

**Overused / Damaging scoring:** If any principle is being used in a way that triggers Reactance (Brehm) — where the visitor feels manipulated, pressured, or deceived — mark it as Overused / Damaging. This is a Fail that outweighs any benefit from the principle's presence.
