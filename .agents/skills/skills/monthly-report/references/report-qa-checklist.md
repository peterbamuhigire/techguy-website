# Report QA Checklist

Run this checklist before sending every report. A report that goes out with errors or vague commentary damages trust more than no report at all.

## Data Accuracy

- [ ] All numbers match the source (GA4, Search Console, email platform)
- [ ] Percentage changes calculated correctly: ((This month - Last month) / Last month) × 100
- [ ] No division errors (especially conversion rate: leads ÷ sessions × 100)
- [ ] "Same month last year" comparison used where available (more meaningful than month-over-month for seasonal businesses)
- [ ] All keyword positions verified against actual Search Console or ranking tool data

## Commentary Quality

- [ ] Every metric has a plain-English sentence explaining what it means — no numbers left unexplained
- [ ] Every decline has: (1) a cause named, (2) an action being taken
- [ ] No jargon left unexplained (if you use "CTR", define it)
- [ ] Executive Summary has exactly 3 bullets — no more, no less
- [ ] "What We Did" section explains WHY each action was taken, not just WHAT was done
- [ ] "Next Month Plan" has specific expected outcomes, not vague goals ("increase organic traffic by 15%" not "improve SEO")

## Tone Check

- [ ] Confident but not arrogant — results are presented without boasting
- [ ] Honest but not defensive — declines are named without excuses
- [ ] Specific not vague — every statement can be traced to a data point
- [ ] Client-centric — written from the perspective of what the client cares about (leads, revenue, growth) not what the agency did
- [ ] No filler phrases: "We are pleased to report...", "As you can see...", "It goes without saying..."

## Completeness

- [ ] Client name correct (double-check — reports get templated and names get left wrong)
- [ ] Month and year correct in header
- [ ] All sections present: Executive Summary, Traffic, Conversions, SEO, Work Done, Next Month, Needs from You
- [ ] "What we need from you" section filled (never blank — always at least "Nothing this month, we have everything we need")
- [ ] Next report date included in footer

## Before Sending

- [ ] Account manager has read the full report (not just skimmed)
- [ ] Any sensitive findings discussed with the client verbally before the written report is sent (no client should be surprised by bad news in a written report)
- [ ] PDF or formatted version created (not raw markdown pasted into email)
- [ ] Calendar invite for monthly review call sent alongside the report
- [ ] Report filed in client folder for year-end review

## After Sending

- [ ] Note in CRM: report sent on [date], call scheduled for [date]
- [ ] If client does not acknowledge within 3 business days: follow up with a single-line email: "Just checking this reached you — let me know if you have any questions before our call."
- [ ] Update client health scorecard (see agency-client-retention skill)

## Red Flags — Do Not Send Until Resolved

- Any metric showing decline for 3 consecutive months without a documented action plan
- Conversion rate below 1% with no explanation and no fix planned
- SEO section with all keywords declining and no cause identified
- "What We Did" section with fewer than 3 items (suggests insufficient work was done — fix delivery before reporting)
- Executive Summary bullet 2 ("watching") that names a problem but gives no action in bullet 3
