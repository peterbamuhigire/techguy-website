# Page Goal Matrix

Use this matrix before page production starts.
Every page must earn its place through business logic, proof logic, and search intent.

## Rule

Each page should have:

- one primary audience
- one primary conversion goal
- one proof burden
- one CTA hierarchy
- one SEO intent

If a page tries to serve too many audiences or too many goals, split it or reduce it.

## Matrix Template

| Page | Primary audience | Buyer stage | Primary goal | Proof burden | Primary CTA | Secondary CTA | SEO intent | Required modules | Notes |
|---|---|---|---|---|---|---|---|---|---|

## How To Fill It

### Primary Audience

Who this page is mainly for.
Not "everyone."

### Buyer Stage

Pick the dominant stage:

- problem aware
- solution aware
- comparison
- decision
- post-purchase trust

### Primary Goal

What the page is supposed to achieve.

Examples:

- book a consultation
- qualify fit
- explain an offer
- prove credibility
- rank for a key intent

### Proof Burden

What the page must prove to deserve its CTA.

Examples:

- expertise
- legitimacy
- results
- process clarity
- fit for a sector

### CTA Hierarchy

Set the order clearly.
If the page is early-stage, the softer CTA may deserve more prominence.

### SEO Intent

What the searcher is trying to accomplish.
Do not use keywords without naming the intent.

### Required Modules

List only modules justified by the page's job.

Examples:

- hero
- trust strip
- process steps
- case study block
- comparison section
- FAQ
- CTA panel

## Quality Checks

- [ ] The page has one dominant audience
- [ ] The page goal matches the CTA
- [ ] The proof burden is visible in the layout
- [ ] The SEO intent matches the page purpose
- [ ] Modules exist because of strategy, not habit
