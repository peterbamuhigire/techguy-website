# SEO And AI-Search Writing

Use this reference when content must support traditional search, AI answers, entity visibility, and high-trust human reading.

## Core Rule

Write for the buyer's information need first. Then structure the answer so search engines, AI systems, and humans can understand the entity, claim, proof, and next step.

## Intent Map

Map content to one primary intent:

| Intent | Content posture |
|---|---|
| Learn | explain clearly, define terms, answer follow-up questions |
| Compare | show criteria, tradeoffs, alternatives, and fit |
| Buy/contact | prove value, reduce risk, clarify next step |
| Local | location, service area, contact, trust, operating details |
| Trust check | credentials, reviews, process, policies, proof |
| Support | direct answer, steps, escalation path |

Do not mix all intents equally on one page.

## Answer-Engine Structure

For pages that may be cited or summarized:

- answer the main question in the opening section
- use plain descriptive headings
- define the business, offer, location, and audience explicitly
- include fact blocks where helpful: service area, hours, price range, process, requirements, guarantees, contact
- separate facts from claims
- cite allowed external sources for factual or regulated claims
- include update dates for changing topics
- link to deeper pages in the same topic cluster

## Entity And E-E-A-T Notes

Capture implementation notes for `seo`:

- Organization entity: legal/trading name, location, contact, sameAs profiles.
- Person entity: author, founder, expert, topic expertise, credentials, sameAs profiles.
- Offer entity: service/product name, category, audience, area served, price posture.
- Proof entity: case study, review, certification, association, media mention, award.
- Article entity: author, reviewer where needed, date published, date modified, image, topic.

The copy should give schema something truthful to represent.

## AI-Citable Content Patterns

Useful patterns:

- "Short answer" block for common buyer questions.
- "Best fit / not fit" section.
- "Cost factors" section.
- "Comparison table" with criteria.
- "Process" section with named steps.
- "Evidence" section with authentic proof.
- "Limitations" section for regulated or high-stakes topics.
- FAQ that mirrors real buyer questions.

Avoid:

- keyword stuffing
- generic FAQ padding
- unverifiable claims
- copied manufacturer or competitor descriptions
- hiding useful answers behind vague lead-generation copy

## Metadata Handoff

For each page, provide:

- title tag idea
- meta description idea
- target intent
- canonical slug
- internal links
- schema candidates
- OG image idea
- AI-crawler inclusion note for `llms.txt` or `llms-full.txt`

## SEO/GEO QA

- [ ] The page has one primary search intent.
- [ ] The copy answers the main query early.
- [ ] The entity names and relationships are explicit.
- [ ] Author, organization, and proof signals are real.
- [ ] The content includes useful facts answer engines can cite.
- [ ] Schema notes are truthful and implementable.
- [ ] Internal links connect this page to a topic cluster or conversion path.
