# Premium Website Gate

Score every primary template from 0-10. Scores below 8 block premium sign-off.

| Category | What To Verify |
|---|---|
| Business clarity | The offer, audience, credibility, and next action are obvious. |
| Visual quality | Layout, type, color, imagery, and spacing feel deliberate and premium. |
| Conversion flow | The page lowers friction and moves the visitor toward a measurable goal. |
| Content quality | Copy is specific, proof is credible, and claims are supported. |
| SEO/content asset value | Money pages have search intent, proof burden, CTA, internal links, schema, and measurement. |
| Accessibility | Keyboard, focus, contrast, reduced motion, labels, and mobile touch targets pass. |
| Performance feel | Images, fonts, motion, and scripts do not make the site feel heavy. |
| Production consistency | Tokens, components, states, and responsive behavior stay coherent across templates. |
| Agency proof | The shipped work is strong enough to support a premium case study, launch report, and developer reputation. |

## Blocking Defects

- First viewport fails to communicate the brand or offer.
- Main CTA is missing, vague, or visually drowned out.
- Mobile layout clips text, hides actions, or creates horizontal overflow.
- Contrast failure on important text or controls.
- Hero imagery is generic, misleading, unusably cropped, or slow.
- Content is generic, thin, unsearchable, or disconnected from the owner's sales process.
- SEO is limited to metadata with no intent map, schema plan, internal linking, or measurement.
- Pricing, contact, booking, checkout, or lead form creates avoidable uncertainty.
- Primary template looks like a disconnected theme rather than part of the same site.
- The site would weaken the developer's premium positioning if shown publicly.

## Evidence

Store the per-template score, screenshot paths, notes, and required fixes in `reports/design-quality/<template>.md` or the project's quality scorecard.
