# App Landing Page Conversion & Trust Design

Average app landing page conversion rate is 2-5%. Top-performing pages achieve 10-20%. The difference is design that eliminates doubt, demonstrates value, and makes the next step effortless.

## The 3-Second Test

A visitor landing on your app page decides instantly:

- **"What does this app do?"**: One clear headline with benefit
- **"Is this legit?"**: App store badges, user count, reviews, professional design
- **"What does it look like?"**: Screenshot or demo visible above fold
- **"How do I try it?"**: Download/signup CTA prominent and clear

## The Download Funnel

```
[Awareness]        →  [Interest]        →  [Desire]          →  [Action]
Social/search         Hero + screenshots   Features + proof     Download/Signup
Referral              Demo video           Testimonials         App Store link
PR/review site        Feature highlights   Pricing clarity      Free trial start
```

### Page Section Flow
1. **Hero**: Headline + subheadline + CTA + screenshot/video — above fold
2. **Social proof bar**: User count, rating, press logos — immediately below hero
3. **Key benefits**: 3-4 benefit cards with icons — what they get
4. **How it works**: 3-step visual process — how to get started
5. **Feature deep-dive**: Detailed features with screenshots — for researchers
6. **Testimonials**: Named users with photos + specific results
7. **Pricing** (if applicable): Transparent tiers with recommended plan
8. **FAQ**: Address objections directly
9. **Final CTA**: Repeat hero CTA with urgency/social proof

## Hero Section Design

The hero section determines 80% of conversions:

### Must-Have Elements

| Element | Guideline |
|---------|-----------|
| **Headline** | One sentence, benefit-focused, 6-12 words max |
| **Subheadline** | Supporting detail, credibility signal, 15-25 words |
| **Primary CTA** | Large, high-contrast, action text ("Start Free", "Download Now") |
| **Secondary CTA** | Lower emphasis ("Watch Demo", "See Pricing") |
| **Visual** | App screenshot, demo video, or animated mockup |
| **Social proof** | User count, star rating, or "Featured in" logos |

### Hero Layout Options

| Layout | Best For | Mobile |
|--------|----------|--------|
| **Left text + right screenshot** | Most apps | Stack (text top, image below) |
| **Centered text + screenshot below** | Simple apps | Natural stack |
| **Full-screen video background** | Premium/lifestyle apps | Replace with static + play button |
| **Split screen** | Before/after apps | Stack vertically |

## CTA Strategy

### Primary CTA Guidelines

- **Text**: Action verb + benefit ("Start Free Trial", "Download Free")
- **Size**: Minimum 48px height, full-width on mobile
- **Colour**: Highest contrast on page (see app-branding.md)
- **Position**: Above fold, repeated at page bottom, sticky on mobile
- **Friction reduction**: "No credit card required", "Free forever", "2-minute setup"

### CTA Text That Converts

| Low Conversion | High Conversion | Why |
|---------------|-----------------|-----|
| "Submit" | "Start Free Trial" | Specific action + benefit |
| "Learn More" | "See It In Action" | Curiosity + engagement |
| "Sign Up" | "Get Started Free" | Low friction + value |
| "Download" | "Download Free — No Signup" | Removes objections |
| "Buy Now" | "Start 14-Day Free Trial" | Risk reversal |

### Sticky Mobile CTA

```
[Fixed bottom bar on mobile]
┌─────────────────────────────┐
│  [★ 4.8] "Download Free"    │
│  [Primary CTA Button]       │
└─────────────────────────────┘
```
- Appears after scrolling past hero
- Shows app rating + CTA
- 60px height, full width
- Semi-transparent or solid background

## Trust Architecture

### Tier 1: Must-Have (Above Fold)

- **App store badges**: Google Play + App Store (official badges only)
- **User count**: "50,000+ teams" or "1M+ downloads" (real numbers)
- **Star rating**: "4.8★ from 2,340 reviews" with link to reviews
- **Press logos**: "Featured in TechCrunch, Product Hunt" (real mentions only)

### Tier 2: High-Impact (Mid-Page)

- **Named testimonials**: Real users with photo, name, title, company, specific result
- **Video testimonials**: 30-60 second clips, hosted on YouTube
- **Case study snippets**: "How [Company] achieved [Result] with [App]"
- **Security badges**: SOC 2, GDPR, encryption (for B2B/finance)
- **Integration logos**: "Works with Slack, Notion, Google" (if applicable)

### Tier 3: Depth (Lower Page)

- **Detailed case studies**: Full page per success story
- **Comparison table**: Your app vs. competitors (honest, factual)
- **Privacy/security page**: Detailed data handling practices
- **Team/company page**: Who built this, why, credibility

## Pricing Page Design

### Pricing Display Rules

| Rule | Rationale |
|------|-----------|
| **3 tiers maximum** | More causes decision paralysis |
| **Highlight recommended** | Visual emphasis on best-value tier |
| **Free tier if possible** | Removes risk, increases signups |
| **Annual savings visible** | "Save 20% annually" with toggle |
| **Feature comparison table** | Checkmarks in grid, key differences clear |
| **No hidden fees** | All costs transparent |

## App Store Optimisation (ASO) Alignment

The landing page should reinforce App Store presence:

- **Same screenshots** used on landing page and store listing
- **Same headline/tagline** for brand consistency
- **Reviews cited** should be verifiable on the store
- **Deep links** to app store listings (platform-aware: iOS → App Store, Android → Play Store)

## A/B Testing Strategy

### What to Test First

| Priority | Element | Test Variants |
|----------|---------|---------------|
| **1** | Hero headline | Benefit vs. feature vs. social proof |
| **2** | CTA text | Action variations ("Start Free" vs "Try Now") |
| **3** | Hero visual | Screenshot vs. video vs. illustration |
| **4** | Social proof position | Above fold vs. below hero |
| **5** | Pricing page | 2 tiers vs. 3 tiers, annual toggle |

### Metrics That Matter

| Metric | Good | Great | Investigate If |
|--------|------|-------|----------------|
| **Bounce rate** | <50% | <35% | >60% |
| **CTA click rate** | >3% | >8% | <2% |
| **Signup/download** | >5% | >12% | <3% |
| **Time on page** | >45s | >90s | <20s |
| **Scroll depth** | >60% | >80% | <40% |

## Common Mistakes

1. **Feature-first hero** — leads with tech specs instead of user benefit
2. **No social proof above fold** — missing the fastest trust signal
3. **Vague CTA text** — "Get Started" with no context of what happens next
4. **Too many CTAs** — competing actions confuse visitors
5. **Missing app screenshots** — visitors need to see the product
6. **Hidden pricing** — "Contact us for pricing" kills consumer app conversions
7. **Desktop-first design** — 70%+ of app landing page traffic is mobile
8. **No FAQ section** — unanswered objections prevent downloads
