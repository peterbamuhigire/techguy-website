# Healthcare Sector Templates

Design strategy for hospitals, clinics, medical practices, and health service websites. Enhanced with clinical-grade UI patterns adapted from healthcare application design.

## Files

| File | Purpose |
|------|---------|
| **sector-brief-template.md** | Customisation questionnaire (REQUIRED) |
| **design-tokens.md** | Clinical color system, typography, spacing (DO NOT USE AS-IS) |
| **component-patterns.md** | HTML/Tailwind component examples |
| **component-patterns-full.md** | Extended component library |
| **IMPLEMENTATION-GUIDE.md** | Step-by-step build guide |
| **premium-hospital-playbook.md** | **$5K+ hospital website playbook** — multi-audience architecture, provider directory, condition pages, booking flow, patient portal showcase, telemedicine, competitive analysis |
| **healthcare-branding.md** | Clinical color system, semantic colours, colour-blind safety, accessibility-first design, typography by facility type |
| **patient-trust-design.md** | Patient funnel, trust architecture, doctor profiles, provider directory, appointment booking, patient portal showcase, telemedicine |
| **healthcare-type-patterns.md** | 9 types (hospital, specialist, wellness, paediatric, mental health, dental, diagnostic, teaching, pharmacy) + premium feature matrix |

## Usage

### For Standard Healthcare Sites ($1-2K)

1. Read **healthcare-branding.md** for design psychology and accessibility
2. Read **patient-trust-design.md** for trust and conversion patterns
3. Read **healthcare-type-patterns.md** to find your facility type
4. Copy **sector-brief-template.md** to `docs/sector-brief.md` and customise
5. Run the design-system skill

### For Premium Hospital Sites ($5K+)

1. **Start with premium-hospital-playbook.md** — follow the complete checklist
2. Read ALL reference files (branding, trust, type-patterns)
3. Complete competitive analysis framework
4. Ensure multi-audience navigation, provider directory, condition pages
5. Copy **sector-brief-template.md** and customise with premium features
6. Run the design-system skill

## Key Differentiators

This healthcare skill incorporates clinical-grade design knowledge:

- **Semantic color system** adapted from EMR/EHR application design
- **Colour-blind safety rules** with triple redundancy (colour + icon + text)
- **Patient portal showcase patterns** for marketing healthcare technology
- **Provider directory specifications** (the #1 revenue-generating page)
- **Multi-audience architecture** (patients, families, physicians, recruits, donors)
- **Condition-specific landing pages** (SEO strategy for 200+ keywords)
- **Premium feature matrix** showing which features matter per facility type
- **Accessibility audit checklist** exceeding WCAG AAA for body text

Healthcare sites must be WCAG AAA for body text with 48px minimum touch targets. Accessibility is mandatory, not optional.
