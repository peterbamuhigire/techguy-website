# Repurposing And Launch Support

Use this reference when social media must support the website as a traffic engine, authority engine, or launch engine.

Primary upstream skills:

- `blog-writer`
- `launch-campaigns`
- `sales-copywriting`

## Core rule

Social media is not a separate content universe.
It should distribute, translate, and intensify the strongest website content and campaign assets.

## Repurposing stack

Start with one strong source asset:

- blog article
- case study
- service page
- webinar page
- waitlist page
- launch page

Then derive platform-native assets from it:

- LinkedIn post
- thread
- carousel
- Reel or short talking-head script
- quote card
- story sequence
- WhatsApp broadcast snippet

## Platform roles in launch support

### LinkedIn

- thought-leadership framing
- proof snippets
- professional CTA to page, event, or waitlist

### Instagram

- carousels from guides
- Stories for reminders and deadlines
- reels for founder explanation or behind-the-scenes

### X / Twitter

- commentary
- quick launch updates
- objection-handling threads

### WhatsApp

- reminder nudges
- event confirmations
- waitlist and launch-day alerts

## Launch content sequence

For timed campaigns, social should usually follow this order:

1. problem agitation
2. education and framing
3. proof
4. offer reveal
5. objection handling
6. close reminder

Each post should point back to the correct website asset for that stage.

## CTA posture by content type

- awareness post -> `Read the guide`
- proof post -> `See the case study`
- consideration post -> `See how it works`
- launch post -> `Join the waitlist`, `Register`, `Book`, or `Apply`

## Guardrails

- Do not post empty engagement bait with no website or offer role.
- Do not launch from social alone when the website page is weak or missing.
- Do not copy the same caption to every platform.
- Do not let social posts promise urgency the website does not support.

## Output standard

A social plan tied to a website or campaign should specify:

- source asset
- repurposed assets by platform
- stage in the sequence
- CTA destination URL
- proof or hook used in each post
