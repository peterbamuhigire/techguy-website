## Critical Issues (Fix Immediately)

### 1. ZERO Structured Data (JSON-LD) — Score: 0/10

Not a single `<script type="application/ld+json">` block exists on any page. This is the single biggest SEO deficit. Google uses structured data for:
- Knowledge Panel (ProfessionalService/Person schema)
- Rich snippets in search results (BreadcrumbList)
- Service carousel (Service schema)
- Local business pack (LocalBusiness with geo coords)

**Impact:** Google cannot understand the entity, services, or location. No chance of appearing in rich results or local pack.

**Fix:** Add to BaseLayout.astro:
- `ProfessionalService` schema on all pages (name, url, logo, address, telephone, email, areaServed, sameAs)
- `BreadcrumbList` schema on all pages
- `Service` schema on services page (4 services)
- `Person` schema on about page

### 2. NO Canonical Tags — Score impact on Technical SEO

Zero `<link rel="canonical">` tags on any page. Without canonical tags:
- Google may index duplicate content (/ vs /en/)
- Ranking signals are diluted across duplicate URLs
- The sitemap already lists both `/` and `/en/` as separate pages

**Fix:** Add `<link rel="canonical" href="https://techguypeter.com/{lang}/{page}/">` to every page in BaseLayout.astro.

### 3. NO Geo Meta Tags — Score impact on Local SEO

Zero geo meta tags (geo.region, geo.placename, ICBM) on any page.

**Fix:** Add to BaseLayout.astro:
```html
<meta name="geo.region" content="UG-102">
<meta name="geo.placename" content="Kampala">
<meta name="geo.position" content="0.3476;32.5936">
<meta name="ICBM" content="0.3476, 32.5936">
```

### 4. No Location Keywords in Page Titles

"Kampala" and "Uganda" do not appear in ANY page title. For local search queries ("IT consultant Kampala", "software developer Uganda"), this is critical.

## High Priority (Fix This Week)

### 5. Meta Descriptions Missing Target Keywords

Most descriptions lack the specific service keywords people search for. Only the contact page mentions "Kampala".

**Fix:** Update descriptions to include:
- Target keywords: "IT consultant", "software developer", "website designer", "ERP systems"
- Location: "Kampala", "Uganda", "East Africa"
- Differentiator: "15+ years", "10+ African countries"

### 6. Broken Font Preload Paths

Two `<link rel="preload">` tags reference `/fonts/playfair-display-latin-700-normal.woff2` and `/fonts/dm-sans-latin-400-normal.woff2`, but no `/fonts/` directory exists in dist/. Fonts are served from `/_astro/` with hashed filenames. These preloads 404 silently.

**Impact:** LCP 100-300ms slower than optimal.
**Fix:** Either remove the preload tags or update paths to match actual `/_astro/` file paths.

### 7. Sitemap Lists Duplicate Root URL

The sitemap lists both `https://techguypeter.com/` and `https://techguypeter.com/en/` — these serve the same content (root redirects to /en/). This signals duplicate content to Google.

**Fix:** Exclude the root `/` from the sitemap, or add a canonical tag on root pointing to `/en/`.

## Medium Priority (Fix This Month)

### 8. Blog Pages Have "Coming Soon" Placeholder Content

Blog pages contain 4 "Coming Soon" placeholder article cards with gradient placeholder images. Word count is only ~255 words (lowest of all pages).

**Impact:** Thin content signals. Google may flag as low-quality.
**Fix:** Either publish real blog articles or add `<meta name="robots" content="noindex">` to blog pages until content is ready.

### 9. Footer Services Not Linked

The 4 services listed in the footer (Software Development, ICT Consulting & Strategy, Business Consulting, Property Management Tech) are plain text, not links.

**Fix:** Convert to anchor links to `/en/services/#software-development` etc.

### 10. Blog Missing From Footer Quick Links

The footer "Quick Links" section lists Home, Services, Portfolio, About, Contact — but not Blog.

### 11. Portfolio Images Missing srcset

Portfolio product screenshots (133-378 KB each) do not use responsive `srcset`. Mobile users download full-size images unnecessarily.

## What's Working Well

- **Performance: 9/10** — Total page weight ~198 KB gzipped. Well under 500 KB target. Zero external HTTP requests. All fonts self-hosted with font-display: swap.
- **Mobile: 9/10** — Viewport meta on all pages. Responsive grid layout. Alpine.js mobile menu with proper ARIA. 16px+ base font. No horizontal scroll.
- **Accessibility: 10/10** — All images have descriptive alt text. Skip-to-content link. Focus-visible styles. Correct lang attribute. ARIA labels on all icon buttons.
- **Security: 10/10** — All external links have rel="noopener noreferrer". Zero mixed content. No external scripts. All internal URLs use relative paths.
- **Internal Linking: 8/10** — Every page reachable in 1 click. No orphan pages. Full nav in header and footer. Strong CTA placement on every page.
- **Heading Hierarchy** — Every page has exactly 1 h1 and sequential h2 > h3 structure. No skipped levels.
- **Hreflang** — Properly implemented with en, fr, and x-default on all pages.
- **Open Graph & Twitter Cards** — Present and correctly configured on all pages.
- **Image Optimisation** — All images converted to WebP by Astro. Hero images eager-loaded, rest lazy-loaded. Width/height attributes prevent CLS.
